- ★ 핵심 함정 준수: app.mode 를 **application.yml** 에 둔다(etc.properties 금지). SSR 페이지 컨트롤러의 @ConditionalOnProperty 가 @PropertySource(etc.properties) 값을 로드 순서상 못 읽어 404 나는 함정(head 가 겪음).
- 조치:
  · application.yml 최상위에 `app.mode: ${APP_MODE:spa}` 추가(head 는 ssr 기본이나 cleanhub 는 spa).
  · WebConfig: `@Value("${app.mode:spa}")` 필드 + addResourceHandlers 앞 `if(!"spa".equals(appMode)) return;`(ssr 이면 SPA 폴백 미등록). isBackendPath 에 `auth/` 보완(없는 /auth/xxx 가 SPA HTML 200 되던 것 → 404).
  · SessionSecurityConfig: `@Value` 필드만 추가(현재 URL 규칙·예외처리는 이미 spa 동작이라 유지, ssr 분기는 SSR 도입 시).
- 변경 파일: `application.yml`, `config/WebConfig`, `config/SessionSecurityConfig`
- 검증: compile SUCCESS. 실기동 spa 스모크 — / SPA 200, 없는 /api·/auth → 404(SPA HTML 아님), /api/admin 미인증 401 JSON, /admin/clients 딥링크 SPA 200. app.mode 는 application.yml 에서 로드 확인.

### [기능] 자동로그인 — refreshToken을 HttpOnly 쿠키로 전환 (세션 앵커 + JWT API + refresh 쿠키)
- 증상: JWT access·refresh 를 프론트 메모리에만 보관 → 새로고침·브라우저 재시작 시 소실 → 관리자 페이지에서 로그인 화면으로 튕김.
- 원인: 토큰 보관이 메모리(JS 변수)라 페이지 리셋 시 사라짐. 라우터 가드는 토큰 없으면 로그인으로 리다이렉트.
- 조치: 인증 구조를 "세션(HttpOnly, 브라우저 열린 동안 로그인상태) + access(JWT, 메모리, /api Bearer) + refresh(HttpOnly 쿠키 + DB, 자동로그인 장기 신분증)"로 정리.
  · 백엔드: `RefreshTokenCookie`(support) 신설 — HttpOnly·SameSite=Lax·Secure(설정값)·Path=/, rememberMe=true면 Max-Age=7일/false면 세션쿠키. `/auth/api/token`은 refresh를 쿠키로 발급(body엔 access·username·roles만, `AuthResult` DTO). `/api/auth/refresh`는 @CookieValue로 refresh 읽어 DB대조+파싱검증→새 access(회전 시 새 쿠키), **세션 불필요=자동로그인**. `/api/auth/logout`은 쿠키 Max-Age=0 삭제+DB삭제. etc.properties `jwt.refresh-cookie.secure`(운영 env REFRESH_COOKIE_SECURE).
  · 프론트: axios `withCredentials`(쿠키 전송), authService 쿠키기반(refresh body 제거), auth 스토어 refreshToken 메모리 제거·rememberMe 기억·`restore()`/`applyAuth`, main.js 앱시작 시 restore()로 복원(라우터 가드보다 먼저), 관리자 로그인 화면 "로그인 유지" 체크박스.
- 변경 파일: `auth/support/RefreshTokenCookie`·`auth/dto/AuthResult`(신설), `auth/controller/AuthApiController`·`TokenRefreshController`, `etc.properties`, `frontend/vue/src/{main.js,plugins/http/axios.js,services/auth/authService.js,stores/auth/auth.js,views/admin/AdminLoginView.vue}`
- 검증: 실기동(:7071) E2E — 발급 시 HttpOnly Set-Cookie(body에 refresh 없음), 세션 없이 쿠키만으로 갱신 성공(재시작 시나리오)+회전 재발급, 갱신 access로 /api/admin 200, 로그아웃 시 쿠키 삭제+DB삭제→이후 갱신 401, 쿠키 없이 갱신 401. vite build SUCCESS.
